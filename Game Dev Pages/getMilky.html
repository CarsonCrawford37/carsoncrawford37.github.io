Get Milky

Role: Programmer
Engine: Unity 3D
Employment: Boise State University (Student)
Team Size: 4

About

Get Milky is a local multiplayer, hide-and-seek game where there are two players: the cowboy and the cow. Setting place on a farm, the cowboy is trying to seek the cow amongst the herd - essentially trying to find a needle in a haystack. The player who is the cow looks just like all the other cows in the herd, requiring the player to be focused on their position all while looking out for the other player. This causes mass confusion for both players, enabling a fun and hectic experience. We also aimed to make this game as inclusive as possible, so we created a custom controller for individuals with limited motor capabilities. Using an Arduino Uno, we conceived, designed, and successfully implemented the controller into our game.

My Contributions

I worked as the main functionality programmer, developing all the game mechanics such as intuitive movement controls, AI state machines, round systems, and multiplayer. I also helped conceptualize the game by including features like the isometric view and hide-and-seek aspect.

Movement -
Movement is everything in games. In this project, we wanted a very specific feel to how the players run around the pen, almost like a slippery feel for increased chaotic gameplay. For this, I used the player’s rigid body and added force when they moved, simulating the sliding feel. With multi-controller compatibility in mind (including the custom Arduino controller), I used Unity’s input manager to gather any input and register it as movement.

AI -
As a key feature, the AI acts as a moving barrier for the players, creating a more engaging gameplay. For this, I used navmesh agents and a simple state machine to bring them to life. The state machine only had two states (sleeping and working) however, it was equipped for many more actions. This just had the AI walk to a random point, stop for a random amount of time, and then walk to another random point to start over. We also wanted to randomly spawn the cows inside the pen, so I created a min and max bounds to keep them within the pen and then spawned them individually.

Round System
This was the main meat of the game that determined the win conditioning. I programmed this system so that if the timer ends before the cowboy catches the cow, then the cow wins, and vice versa.

Multiplayer
I used Unity’s Simple Multiplayer package to allow multiple players to join via the same machine. I also designed an auto-player prefab assignment script where it will automatically give the player a role based on when they join.
